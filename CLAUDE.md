# Rubolka

## Цель проекта 🎯
Продающий лендинг бренда **RUBOLKA** — производителя одежды оптом со своим производством в Москве. Страница превращает холодный трафик в заявки: показывает преимущества собственного производства, ассортимент (футболки, худи, мерч, спортивная линейка), виды трикотажных полотен, варианты нанесения принта, отзывы и условия доставки, и ведёт пользователя в Telegram (@rubolka_opt) и WhatsApp для оформления оптового заказа.

## Побочные цели
- Витрина ассортимента и качества: каталог изделий и образцов полотен (`assets/wear`, `assets/production`, `animation/`).
- Социальное доказательство: блок отзывов (`assets/reviews`) и ссылки на Telegram-канал.
- Демонстрация производственного цикла «пошив → печать → упаковка» через hero-видео и анимацию на скролле (`animation/VIDEO_PROMPTS_v2_gen4.md`).
- Рабочая площадка для итераций креатива и контента (`uploads/`).

## Ценности проекта
- Своё производство и контроль качества как главный аргумент отстройки.
- Конкретика вместо общих слов: реальные фото изделий, полотен и отзывов.
- Скорость: посетитель доходит до контакта (Telegram / WhatsApp) за минимум шагов.
- Тёмная премиальная эстетика бренда: единый фон `#0D0D0D`, акцент `#E8001D`, шрифты Oswald / Inter / JetBrains Mono.

## Архитектура
Статический одностраничный лендинг без бэкенда и без системы сборки — открывается как обычный HTML-файл.

**Страницы**
- `index.html` — единственная боевая страница (весь CSS и логика встроены inline).
- `Rubolka Landing.html` — идентичная копия `index.html` (рабочий дубликат/резерв).
- Секции страницы: «Почему мы» → ассортимент / полотна → «Нанесение» (печать) → «Начать работу» → «Отзывы» → «Доставка» → финальный CTA в Telegram / WhatsApp.

**Assets**
- `assets/` — боевые медиа лендинга: `favicon.jpg`, `logo*.jpg`, `wear/` (изделия и мерч), `production/` (производство), `reviews/` (скрины отзывов), `tg-channel-*.png`.
- `animation/` — исходники и промпты hero-видео/анимации (Runway Gen-4), референс-картинки полотен (Рибана, Кашкорсе, Двухнитка и др.).
- `uploads/` — рабочая папка сырых материалов (фото, черновики, PDF). Содержит пользовательские/исходные данные — коммитить выборочно, не пакетно.
- `Rubolka.zip` — крупный архив, исключён из git через `.gitignore` (`*.zip`).

**Деплой**
- Конфигов сборки/CI/CD нет. Это набор статических файлов: деплой = выложить `index.html` + папки `assets/` и `animation/` на любой статический хостинг (GitHub Pages / Netlify / Vercel / nginx).
- Remote: `https://github.com/AgentSmoki/rubolka.git`, ветка `main`.
- Внешние зависимости: Google Fonts (CDN). Контакты-выходы: Telegram `t.me/rubolka_opt`, WhatsApp `wa.me/79370958752`.

## Prompt Caching
Перед проектированием системного промпта, message-массива, RAG-пайплайна или любой архитектуры запросов к LLM — **обязательно** прочитай скилл `~/.claude/skills/prompt-caching-playbook/SKILL.md`.

Норма cache hit rate: **80–90%**. Правило большого пальца: статика → в начало (tools → system → long context → history), динамика (timestamp / UUID / session_id / trace_id) → в самый конец, после последнего cache breakpoint. Никогда не помещай меняющиеся данные перед статическим контекстом.

Триггер на скилл: любой вопрос о cache hit rate, `cache_control`, `prompt_cache_key`, Context Caching API, cache TTL, "почему API дорого", "как уменьшить токены", cache miss, kv-cache, prefix matching.
